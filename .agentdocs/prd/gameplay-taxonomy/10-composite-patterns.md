# 复合玩法模式

## 定位

真实课程游戏通常不是单一玩法，而是多个基础玩法串联。复合玩法的关键不是堆玩法，而是让每一步服务同一个阶段目标，并让前一步产物成为后一步输入。

## 常用复合链

### 观察 + 分类

流程：先看材料，再把对象分类。

适合：设备识别、数据清洗、风险归类、生物分类。

示例结构：

1. 场景出现一批混杂对象。
2. 学生先扫描或点击查看细节。
3. 再拖拽或盖章分类。
4. 系统按类别恢复不同模块。

### 标注 + 路径

流程：先标出关键线索，再让路线或系统状态变化。

适合：地图任务、Attention、阅读抓重点、地理路线。

示例结构：

1. 文本或地图给出多个线索。
2. 学生框选或点击关键部分。
3. 地图路线、流程路径或权重图实时改变。
4. 错标干扰项会让路线偏航。

### 搜证 + 裁决

流程：先找证据，再做判断。

适合：事实核验、历史材料、AI 安全、责任归因。

示例结构：

1. 给出一个看似合理的结论。
2. 学生翻材料找证据。
3. 把证据链拼完整。
4. 最后放行、打回或追责。

### 调参 + 对比

流程：先改变量，再比较结果。

适合：科学实验、Prompt、模型训练、物理参数。

示例结构：

1. 给出初始结果和目标。
2. 学生调整一个或多个参数。
3. 运行得到新结果。
4. 对比版本并总结规律。

### 拼装 + 测试

流程：先搭系统，再运行测试。

适合：Agent、工作流、编程、神经网络、工程结构。

示例结构：

1. 系统缺少模块或连接。
2. 学生装配节点、模块或规则。
3. 点击运行测试。
4. 根据失败日志继续修复。

### 创作 + 验收

流程：先产出作品，再按标准检查并迭代。

适合：项目课、视频、Skill、Vibe Coding、写作。

示例结构：

1. 学生配置或生成作品。
2. 系统展示成品。
3. 学生按验收清单找问题。
4. 修改输入或配置后生成新版本。

## 按教学阶段组合

### 新概念导入

推荐组合：观察发现 + 单选判断 + 短反馈。

目标是让学生看见现象，不要让规则太复杂。

### 概念辨析

推荐组合：分类分流 + 错误归因 + 回放解释。

目标是暴露边界误区。

### 机制理解

推荐组合：拼装建构 + 流程接线 + 运行测试。

目标是让学生看到系统如何运作。

### 熟练巩固

推荐组合：反应街机 + 错题回炉 + 分波挑战。

目标是提高速度和准确率。

### 高阶迁移

推荐组合：搜证推理 + 策略决策 + 作品验收。

目标是让学生把知识用于真实任务。

## 选型检查清单

设计复合玩法前确认：

- 每一步是否对应同一个阶段目标。
- 前一步结果是否会影响后一步，而不是并排小游戏。
- 错误反馈是否能说明具体误区。
- 是否有可见的系统状态变化。
- 是否能在课堂时间内完成。
- 是否能写成清晰状态机。
