# 课程游戏玩法超类索引

## 定位

本目录用于把通用课程游戏玩法按“学生核心动作”拆成多个超类文件。设计任何学科、主题或知识点的游戏化教学前，先读本索引判断应该进入哪个超类，再阅读对应文件挑具体玩法。不要按学科名称固定套玩法；同一学科内部会因为知识形态、学习深度和目标动作不同而需要完全不同的玩法。

这里的分类目标不是穷尽市面所有游戏名，而是覆盖学生学习知识时可能需要做的核心认知与操作动作。具体游戏可以无限变化，但通常都能拆回这些超类。

## 玩法选择原则

玩法由学习任务决定，而不是由学科决定。设计时先判断学生这次是在认识概念、练熟技能、理解结构、推理证明、探索变量、表达沟通、排错验证、做价值判断，还是完成作品。学科只提供素材和语境，不直接决定玩法。

同一个知识点也可以按学习深度切换玩法：

- 初识：观察发现、单选判断、分类分流。
- 理解：匹配配对、标注编辑、流程排序、结构拼装。
- 练熟：射击命中、接落物、快问快答、错题回炉。
- 迁移：搜证推理、策略决策、诊断修复。
- 创作：模板生成、素材编排、工作流搭建、作品验收。

## 覆盖性检查

按学习任务看，常见知识都能先落到以下入口：

- 概念边界：信息识别与组织、选择判断、分类分流。
- 事实记忆：动作熟练与即时反应、匹配配对、记忆复现。
- 结构组成：数量空间与系统建构、拼图拼装、模块装配。
- 步骤流程：序列逻辑与推理、流程接线、复合玩法。
- 数量关系：数量空间与系统建构、实验模拟与变量调参。
- 变量规律：实验模拟与变量调参、A/B 对比、沙盒模拟。
- 逻辑证明：序列逻辑与推理、搜证推理、证明步骤补全。
- 表达沟通：语言表达与沟通、角色扮演、创作生产。
- 错误修复：诊断调试与验证、回归测试、风险闸门。
- 价值权衡：策略经营与角色决策、听证裁决、资源分配。
- 项目产出：创作生产与项目制、作品验收、用户测试。

## 设计流程

1. 写清本小节知识点，不超过一个主要目标。
2. 判断学习深度：初识、理解、练熟、迁移、创作或验收。
3. 判断学生需要做的核心动作。
4. 进入对应超类文件，挑 2-3 个候选玩法。
5. 比较哪种玩法的错误反馈最能暴露误区。
6. 把玩法套入课程剧情、视觉风格、状态机和完成条件。

## 扩展原则

新增玩法时，优先补到已有超类。只有当玩法的核心动作、反馈机制或学习价值无法归入现有超类时，才新增超类文件。
