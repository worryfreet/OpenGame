# 创作生产与项目制类

## 适合什么知识

创作生产与项目制类玩法解决“做出一个作品，并按标准迭代到可用”。它适合项目课、艺术、多媒体、编程、Coze 智能体、Skill、工作流和 Vibe Coding。

不适合：还没有建立基础概念时直接使用。项目制玩法需要配套验收标准，否则学生只会追求表面完成。

## 模板生成

### 字段填充生成

核心动作：填写关键字段生成规范作品。

交互变体：

- 菜单卡。
- 实验报告。
- 学习计划。
- 海报文案。

反馈方式：字段完整度、格式合规、表达质量。

适合：应用文、Skill、结构化输出、低门槛项目。

### 规则模板生成

核心动作：按规则生成多种结果。

交互变体：

- 菜单卡批量生成。
- 题目生成。
- 海报模板换内容。
- 机器人回复格式生成。

反馈方式：一致性、字段漏项、规则违反提示。

适合：Skill、批量内容、结构化 AI 输出。

## 素材编排

### 分镜板

核心动作：把文本或素材拆成镜头。

交互变体：

- 诗句拖到镜头槽。
- 意象卡配画面。
- 情绪曲线。
- 镜头顺序调整。

反馈方式：镜头数量、意象具体度、情绪连贯性。

适合：语文、视频工作流、美术、历史故事。

### 时间轴编辑

核心动作：把图片、视频、音频、字幕放到时间线上。

交互变体：

- 拖素材到轨道。
- 调整时长。
- 字幕对齐。
- 合成预览。

反馈方式：重叠警告、空白片段、节奏评分。

适合：视频项目、音乐、演示文稿、多媒体创作。

### 画布排版

核心动作：在画布上安排元素。

交互变体：

- 海报布局。
- 菜单卡布局。
- UI 页面搭建。
- 展板设计。

反馈方式：对齐、层级、可读性、视觉焦点。

适合：美术、信息设计、网页、项目展示。

## 规则造物

### 游戏规则描述

核心动作：用自然语言描述可运行游戏。

交互变体：

- 填游戏需求表。
- 规则卡拼装。
- 胜负条件配置。
- 难度波次设计。

反馈方式：需求完整度、歧义提示、可实现性评价。

适合：Vibe Coding、游戏设计、编程启蒙。

### 智能体设计

核心动作：配置角色、规则、知识和输出格式。

交互变体：

- System Prompt 配置器。
- 知识库绑定。
- 测试对话。
- 约束调整。

反馈方式：回复质量、是否越界、是否引用知识。

适合：Coze 智能体、Agent、AI 应用。

### 工作流搭建

核心动作：把节点接成可运行流程。

交互变体：

- 开始节点。
- LLM 节点。
- 工具节点。
- 循环与条件节点。

反馈方式：运行日志、节点失败、数据结构检查。

适合：Coze 工作流、多媒体生成、自动化任务。

## 作品验收

### 需求清单验收

核心动作：按要求逐项检查作品。

交互变体：

- 勾选验收项。
- 缺陷标注。
- 发布前审核。
- 修复建议。

反馈方式：通过 / 待修 / 不可发布，附具体原因。

适合：项目课、编程、AI 生成物、应用文。

### 版本对比

核心动作：比较多个版本优劣。

交互变体：

- V1/V2/V3 对比。
- 差异高亮。
- 选择保留修改。
- 回滚版本。

反馈方式：质量变化曲线、改动影响说明。

适合：写作、Prompt、Vibe Coding、设计迭代。

### 用户测试

核心动作：从用户视角测试作品。

交互变体：

- 模拟用户任务。
- 找 Bug。
- 操作路径记录。
- 体验评分。

反馈方式：失败路径、可用性问题、改进优先级。

适合：网页、游戏、智能体、工具应用。
