# Course Studio 核心诉求保真与反模板化

## 任务背景

用户进一步澄清：上一轮“枪战”只是举例，真正要解决的是所有课程生成都必须贴近用户核心诉求。系统可以排除危险、侵权或不适龄元素，但不能把用户想要的核心体验删掉。Playlet 必须足够精彩，课程要充分发挥 AI 的创造力；稳定模板是底座，不是让用户一眼看出套模板的理由。

## 顶级原则

- 核心诉求保真：先识别用户真正想要的体验、操作、身份、节奏和情绪，再做安全/版权/适龄替换。
- 安全替换只改表象：危险元素、知名 IP、血腥伤害等需要替换，但核心动词和体验必须保留，例如瞄准、经营、建造、侦探推理、探索、竞速、创作、合作等。
- Playlet 精彩度优先：每个 playlet 必须有明确身份、目标、可见状态变化、即时反馈、错因提示和奖励/解锁。
- AI 创造力受控释放：AI 不应只填模板字段；应负责创意包装、关卡节奏、反馈后果、素材 prompt、过场构想和玩法组合，但不能绕过 schema、mapper、runtime 和验证。
- 反模板化红线：课程不能让用户一眼看出是同一套模板换皮。模板负责稳定，AI 负责让每门课在角色、任务、状态、素材、节奏和反馈上有独特性。

## 实施结果

- Course Studio 入口已从“枪战安全改写”扩展为“所有用户偏好都必须核心诉求保真”，要求先提炼身份、操作、情绪、节奏、挑战和世界状态。
- `generate_course_plan` 已要求方案体现 AI 创造力：角色身份、世界问题、核心任务、状态变化、奖励、错因反馈和素材方向必须围绕本次用户诉求定制。
- `gameDirector` 新增 `intentFit` 与 `antiTemplateOriginality` 质量维度，拦截偏好只当表层主题、方案模板味过重的问题。
- 测试覆盖了侦探推理这类非动作偏好被退化成普通问答时的阻断，以及 Course Studio prompt 对所有偏好保真的要求。
- 已通过 core/cli 相关测试、typecheck 和 lint；本任务未重新生成实际课程包，聚焦修复生成链路、质量门禁和长期文档记忆。
