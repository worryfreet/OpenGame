# MVP 3.0 一句话生成导演规则

## 使用场景

开发一句话课程生成、`generate_course_plan` 提示词或 CoursePlanOption 自动修复时阅读。本文只约束方案层，不允许绕过 CourseSpec、CoursePlanOption、Course GDD 和课程包验证链路。

## 导演目标

一句话输入生成的课程方案必须先让学习目标成为玩法规则，再进入模板和素材生成。合格方案不是“主题皮肤 + 选择题 + 答对加分”，而是让学生的概念判断改变路线、资源、角色状态、机关、关卡进度或任务状态。

## CoursePlanOption 必备信号

- `learningLoop` 至少包含情境导入、观察示例、核心操作、即时反馈、复盘评价。
- `scenePlan` 至少体现情境目标、核心挑战、状态推进、迁移挑战或学习报告。
- `workflow` 至少 2 个 ready playlet node，并通过 edge 表达成功、失败或部分正确后的推进。
- `recommendationReason` 必须说明知识点如何影响玩法状态，而不只是说明题目覆盖。
- `assessmentPoints` 覆盖所有 `learningGoals`，但覆盖评价点不等于玩法合格。
- workflow node 的 `goalIds` 使用 `goal_1`、`goal_2` 这类稳定目标编号，学生可见 config 文案使用任务语义，不直接复述完整学习目标。

## 节奏节点

`gameDirector.reviewGameDirection()` 会提取以下节点：

- 情境目标：学生知道自己在世界中要完成什么。
- 观察示例：先看到概念或策略如何起作用。
- 核心挑战：通过拖拽、排序、连接、装配、调参、排查、分配等动作使用知识。
- 状态推进：正确或错误判断改变任务状态、资源、路线、机关或关卡进度。
- 迁移复盘：换情境检查掌握，并进入学习报告。

少于 3 类节奏节点时，方案应被视为导演不完整。

## 换皮问答拦截

以下信号会触发阻断或重写：

- 玩法描述集中在答题、问答、选择题、给分、刷题。
- 方案没有状态变化，只在答对后加分或进入下一题。
- 反馈没有后果，只显示正确或错误。
- 学习目标只出现在 `assessmentPoints`，没有进入 playlet config、场景操作或推荐理由。
- 学生端标题、任务、按钮、场景名或推荐语直接写“学习目标是……”“掌握……”等教学话术。

## 生成提示要求

模型生成方案时必须明确回答三个问题：

- 学生做了什么操作？
- 这个操作基于哪个学习目标或概念判断？
- 判断结果如何改变游戏状态并触发反馈后果？

若无法回答，应重写方案，不进入 Course GDD、素材或 TTS。

## 一句话入口

SDK 可视化入口应优先使用 `createCourseGameFromPrompt()`，只传用户一句话、可选 profile、偏好、学习状态和家长策略。该入口必须先调用 `generate_one_shot_course_plan`，只有拿到受控 `CourseSpec` 后才进入 `generate_course_plan`；追问或阻断状态必须返回给外部 ToC 服务，不允许自行猜测继续生成。
