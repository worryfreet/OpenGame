# MVP 3.0 精彩度评分标准

## 使用场景

开发一句话生成、课程质量门禁、自动修复循环或 golden cases 回归时，必须使用本标准判断课程是否足够精彩。它不替代 CourseSpec、CoursePlanOption 或 Course GDD 校验，而是在受控链路之上评估游戏化课程体验质量。

## 六个维度

| 维度       | 核心问题                                | 低分信号                           | 改进方向                                   |
| ---------- | --------------------------------------- | ---------------------------------- | ------------------------------------------ |
| 目标清晰   | 学习目标是否具体且可观察                | 目标过多、过空，缺掌握证据         | 收敛到 2-4 个目标，为每个目标补掌握证据    |
| 游戏循环   | 是否有观察/操作、状态变化、反馈和再决策 | 只做答题，答对加分但知识不影响状态 | 把知识判断绑定到路线、资源、角色或关卡状态 |
| 惊喜推进   | 是否有解锁、变化、阶段目标或迁移挑战    | 单场景线性讲解，没有关卡变化       | 增加阶段目标、失败回流、升级或新规则       |
| 反馈丰富   | 是否解释错因和下一步                    | 只有正确/错误或只给答案            | 使用步骤化反馈、错因标签和苏格拉底提示     |
| 世界吸引力 | 主题、角色、兴趣和视觉氛围是否统一      | 兴趣只写在文案里，不影响任务       | 把学生兴趣变成角色身份、任务目标和视觉道具 |
| 挑战曲线   | 是否从示例、引导练习到独立挑战和迁移    | 一上来高难度，或全程低压重复       | 按年级和深度安排逐步提升，并提供补救路径   |

## 门槛

- 总分默认最低 75。
- 任一维度默认最低 60。
- 低于门槛的方案不能进入高成本资产生成。
- 缺少 CoursePlanOption 时，只能评估输入潜力；游戏循环和推进节奏应保守计分。

## Golden Cases

`packages/core/src/course/quality/goldenCases.ts` 保存 MVP 3.0 阶段 0 的结构化 golden cases。每个样例必须包含一句话输入、期望 CourseSpec、期望玩法方向、ready playlet、最低精彩度分。

阶段 0 的 golden cases 只做单元回归，不直接训练模型。阶段 6 再把它们接入端到端一句话生成回归。

## 自动修复提示

评分结果必须能转成修复动作，例如：

- 增加一次失败后的提示重试路径。
- 将学习目标绑定到资源变化或关卡解锁。
- 把浅层正确/错误反馈改为错因解释和下一步提示。
- 补充迁移任务，验证学生能在新情境中使用知识。
